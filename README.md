# Content Machine Skills

Скиллы для код-агентов (**Claude Code** / **Codex**), которые превращают идею в готовый контент. Командуешь словами — агент сам идёт в нужный сервис, генерирует и приносит результат.

## Скиллы в наборе

| Скилл | Что делает |
|---|---|
| **storyboard-builder** | Из одной фразы-идеи собирает мини-ролик «как режиссёр»: креативный бриф → ассеты (лицо/товар/локация) → шотлист → промты → кадры-картинки → оживление в видео → склейка в тизер. Держит одно лицо во всех кадрах, делает плавные переходы и не сжигает кредиты — останавливается и спрашивает «ок?» перед каждой тратой. |
| **talking-twin** | Цифровой двойник без камеры: из текста собирает вертикальный ролик 9:16, где твоё лицо говорит твоим голосом. Голос — клон ElevenLabs (силён в русском) или твоё записанное аудио; лицо и синхрон губ — аватар HeyGen. Аудио подаётся звуковой дорожкой, поэтому слышен именно твой голос, а не голос аватара. Перед тратой кредитов показывает остаток и спрашивает «ок?». |

## Установка

```bash
# для Claude Code
npx skills add viktorshab/content-machine-skills --agent claude-code

# для Codex
npx skills add viktorshab/content-machine-skills --agent codex
```

После установки агент подхватывает скиллы автоматически. Запуск — обычной фразой, например:

> **storyboard-builder:** собери раскадровку: я владелец кофейни, эффектный тизер нового сезонного меню, 9:16, 7 кадров, на ключевых кадрах — моё лицо

> **talking-twin:** сделай вертикальный ролик-двойник 9:16 моим голосом и лицом из текста: «…». Сначала превью, чистовое — после моего «ок».

## Что нужно, чтобы скиллы работали

Зависит от скилла — у каждого свой канал, ключи кладутся в `.env` (в чат не пишутся, в git не коммитятся):

- **storyboard-builder** — подключённый **канал генерации** картинок и видео на выбор: **Higgsfield** (через MCP, одной командой, вход в браузере — самый низкий порог) или **Replicate** (по ключу-токену, оплата по факту центами — больше моделей и контроля). Скилл не дёргает API руками — он просит агента сгенерировать через подключённый канал.
- **talking-twin** — два сервиса: **ElevenLabs** (ключ + id клон-голоса) для голоса и **HeyGen** (ключ + id аватара, отдельный API-кошелёк) для лица. Скрипт читает доступы из `.env` сам и собирает ролик — в интерфейсы сервисов заходить руками не нужно.

## Как устроен storyboard-builder

- `SKILL.md` — ядро: дух, 5 законов (image-first, один референс лица, frame chaining, стоп-гейт перед тратой, «не правишь промт руками — даёшь заметки»), конвейер из 8 фаз со стоп-гейтами, методслой и границы.
- `references/01-assets-phase0.md` — ассеты-«паспорта» (лицо/товар/локация/пропсы) и тест связки.
- `references/02-brief-shotlist.md` — креативный бриф → референсы → шотлист.
- `references/03-prompts.md` — формула промта (MCSLA), единый style header, отладочные приёмы.
- `references/04-generate-stitch.md` — генерация кадров, оживление (frame chaining), склейка (ffmpeg), папки проекта.
- `references/05-engines-channels.md` — Higgsfield / Replicate / Flux LoRA, выбор модели, безопасность ключа.

## Как устроен talking-twin

- `SKILL.md` — ядро: дух, маяк качества (в ролике слышен ТВОЙ голос), связка «голос и лицо собираем отдельно», типы аватара (фото / видео), стоп-гейт перед тратой кредитов, этика (двойник только себя), безопасность ключей.
- `scripts/make_twin.mjs` — сборка ролика одной командой: озвучка клон-голосом (ElevenLabs) → загрузка аудио дорожкой → генерация 9:16 (HeyGen) → готовый mp4. Режим `--list` показывает твои аватары и голоса с готовыми id. Тип аватара определяет сам.
- `scripts/fetch_video.mjs` — забрать уже готовый ролик по его `video_id`, если ожидание оборвалось (без повторного расхода кредитов).
- `.env.example` — образец файла с ключами (скопируй в `.env`, впиши свои).

## Благодарности (источники методики)

Скилл собран на основе публичных материалов:
- референс-каркас (папки `attempts/approved/disapproved`, approval-gates, frame anchoring, ffmpeg-склейка) — [Samin12/ai-storyboard-video-starter](https://github.com/Samin12/ai-storyboard-video-starter) (MIT);
- дисциплина промтинга (формула MCSLA, «plausibility is not validity») — [OSideMedia/higgsfield-ai-prompt-skill](https://github.com/OSideMedia/higgsfield-ai-prompt-skill) (MIT);
- identity-lock (Soul ID) и безопасная работа с ключом — [higgsfield-ai/skills](https://github.com/higgsfield-ai/skills) (MIT);
- язык камеры/света — [rediumvex/ai-video-generator-claude](https://github.com/rediumvex/ai-video-generator-claude) (MIT);
- методика конвейера и приёмы продакшена — публичные видео-разборы (Higgsfield + Claude).

Методический слой (стоп-гейты, анти-рационализация, проверка результата «чужими глазами») — авторский.

**talking-twin** работает напрямую через официальные API ElevenLabs (озвучка/клон голоса) и HeyGen (аватар, синхрон губ); подача аудио звуковой дорожкой (чтобы сохранить твой голос), стоп-гейт перед тратой и проверка «чужими глазами» — авторские.
