# Фазы 5–7 — генерация кадров, оживление, склейка + папки проекта

> Здесь раскадровка превращается в файлы. Главное — повторяемость: всё лежит по папкам, к проекту можно вернуться. Источники: репо Samin12 (`workflow.md`, `approval-gates.md`, `stitching-notes.md`, папки `attempts/approved/disapproved`), видео `a6lAPOo4CXA` (агент сам раскладывает по папкам), Reddit-практика (деградация при frame chaining).

## Структура папок проекта (Samin12 + a6lAPOo4CXA)

Под проект — отдельная папка, внутри нумерованный конвейер по фазам. На **каждой** стадии — тройка:
- `attempts/` — именованные попытки-эксперименты (напр. `attempt-1-broadcast`, `attempt-2-kinetic`);
- `approved/` — утверждённое (копируем победителя сюда, не перемещаем);
- `disapproved/` — отклонённое + **одна строка почему** (identity drift / объект изменился / локация «другая комната»).

Плюс `00-admin/approval-log.md` — реестр решений (стадия · артефакт · утверждено? · заметка).

**Где создать и как (готовый шаблон, не выдумывай на ходу).** Агент сам создаёт папку проекта **в текущей рабочей папке проекта** (или в папке, которую назовёт пользователь) — имя `проект_<идея>` (напр. `проект_кофейня-тизер`). Дерево:
```
проект_<идея>/
├── 00-admin/        approval-log.md, project-status.md
├── 00-assets/       паспорта: лицо, локация, товар, пропсы (Фаза 0)
├── 01-brief/        attempts/ approved/ disapproved/
├── 02-references/   attempts/ approved/ disapproved/
├── 03-shotlist/     attempts/ approved/ disapproved/
├── 04-prompts/      attempts/ approved/ disapproved/
├── 05-frames/       attempts/ approved/ disapproved/   ← кадры-картинки
├── 06-videos/       attempts/ approved/ disapproved/   ← оживлённые клипы
└── final-outputs/   склеенный тизер
```
Создай это дерево в начале (Фаза 0), дальше складывай артефакты по фазам.

**Железные правила (Samin12):**
1. **Следующая фаза строится ТОЛЬКО из `approved/`.** От отклонённого ничего не наследуем.
2. **Отклонённое не удаляем** — на нём учится следующая попытка (*«Rejected frames should stay so the next attempt can learn»*).
3. Из видео (a6lAPOo4CXA): агент сам создаёт папку проекта и складывает туда бриф, все промты и итоговые видео — чтобы человек не потерял и мог вернуться, посмотреть какой промт сработал, переиспользовать.

## Фаза 5 — ключевые кадры (картинки)

Генерируешь опорные кадры image-моделью (см. `05-engines-channels.md`). Приёмы консистентности:
- **Контакт-лист** (Samin12 `contact-sheet`): собери сетку всех кадров одной картинкой — так консистентность лица/стиля видна разом, а не по одному.
- **Seed-locking**: у удачного кадра зафиксируй seed/параметры — чтобы итерировать рядом, а не заново вслепую.
- **Точечный re-roll слабых кадров** (Samin12 `identity-extras`): если на одном кадре лицо «слабое» — перегенери ТОЛЬКО его (отдельный identity-first проход + сравнительный лист), а не весь набор.

🛑 **Стоп-гейт:** «кадры консистентны, лицо узнаётся, годятся как якоря для видео?»

## Фаза 6 — оживление в видео (frame chaining технически)

⚠️ **Точка невозврата по кредитам** — запускаешь только после подтверждения промтов и кадров.

**Три способа сшить кадры** (выбираешь под движок и задачу):

**Способ A — чистые keyframes заранее (надёжнее всего против дрейфа).** Сначала генеришь ВСЕ опорные кадры как чистые картинки (включая промежуточные A/B/C для проезда), потом оживляешь каждую ПАРУ соседних: клип1 = кадр A→B, клип2 = кадр B→C. Кадр B берёшь чистый (нарисованный как картинка), а не вытянутый из «поплывшего» видео. Минус — больше картинок заранее; плюс — дрейф почти не копится.

**Способ B — последний кадр предыдущего клипа (быстрее).** Классическая цепочка:
1. Генеришь клип 1 (из опорного кадра).
2. Экспортируешь **последний кадр** клипа 1.
3. Грузишь его **стартовым кадром** клипа 2.
4. Промт клипа 2 — **только движение и камера** (внешность держит стартовый кадр); начни словами `From this exact pose, …`.
Каждые 2-3 клипа возвращайся к чистому опорному кадру, иначе лицо/цвет/объект уплывут.

**Способ C — один multi-shot вместо ручной склейки (Kling 3.0).** Kling умеет Custom Multi-Shot: несколько кадров с длительностями одним промтом (`Shot 1, 2 seconds: …` + общий блок `Continuity:` в конце) — получаешь связную сцену в одной генерации. Но **реальное лицо в close-up и самый динамичный проезд держи отдельными клипами**, чтобы переснять только проблемный кусок, а не весь ролик.

Модели с поддержкой start/end frame (нужно для способов A/B и для direction-critical кадров): Kling 3.0, Veo 3.1, Hailuo, Luma — задаёшь обе крайние точки, модель интерполирует движение между ними и не угадывает направление. Полная шпаргалка моделей — `05-engines-channels.md`.

⚠️ **Предупреждение о деградации (Reddit-практика, обязательно учесть):** при длинной цепочке артефакты **накапливаются** — цвет «уплывает», лицо дрейфует, потому что каждый следующий клип стартует с уже слегка артефактного кадра. Лечение: **подчищай/апскейль стыковочный кадр** перед подачей в следующий клип; не делай слишком длинные цепи без периодического возврата к исходному референсу.

Когда отключать chaining (a6lAPOo4CXA): если нужны независимые кадры — скажи агенту «все кадры только стартфрейм, не соприкасаются». В реальном кейсе видео часть кадров идёт по chaining, часть — независимо, и это нормально.

## Фаза 7 — склейка в один ролик (Samin12 `stitching-notes.md`)

Агент собирает клипы сам — монтировать вручную не нужно. Для склейки нужен установленный **FFmpeg** (бесплатный) — если в системе его нет, поставь сам / попроси агента поставить ещё до этого шага, чтобы не застрять на финале.
- **Где резать (хороший шов):** на вспышке, на быстром движении камеры, на совпадающей позе, на звуковом ударе. Принцип: **«the best transition is usually invisible»**.
- **ffmpeg (concat-демуксер)** — простейшая склейка без перекодирования:
```bash
# concat-list.txt:
# file 'shot-01.mp4'
# file 'shot-02.mp4'
# file 'shot-03.mp4'
ffmpeg -f concat -safe 0 -i concat-list.txt -c copy final-output.mp4
```
- **Если у клипов разные fps/кодеки/аудио** — сначала пере-экспортируй все в один формат, потом склеивай (иначе рассинхрон/артефакты на швах).

🛑 **Стоп-гейт:** «это ощущается как один цельный ролик, а не набор кусков?» Победитель копируется в `final-outputs/`.

## Переиспользование (экономия)

Неподошедшие генерации не выбрасывай: даже забракованный клип отдаёт полезный **последний кадр для chaining** или кадр-референс позы. В видео (ODN) так и делают: *«nothing got wasted»*.
