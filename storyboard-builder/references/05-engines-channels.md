# Паспорт движков и каналов генерации

> Какими движками генерим и через какой канал их вызываем. Канал — **сменное звено**: методика конвейера одна, движок внутри меняется. Все цифры (цены, лимиты, кредиты) — **сверять с доками на день работы**, они меняются часто (особенно Higgsfield).

## Как вызываем движки

Генерацию вызывай через **свой подключённый канал** — Higgsfield (MCP) или Replicate (по ключу), как настроено в уроке. Скилл не дёргает API руками: он просит агента сгенерить через подключённый канал, агент сам идёт в нужную модель.

## Два канала (выбираешь под себя)

### Канал 1 — Higgsfield (готовое «под ключ»)
- Подключение одной командой, вход через браузер (OAuth), без ключей:
  `claude mcp add --transport http --scope user higgsfield https://mcp.higgsfield.ai/mcp`
- За одной точкой — 30+ моделей; командуешь «сгенери в такой-то» → агент маршрутизирует.
- **Soul ID** — обучаешь своё лицо (~20 фото, обучение пара минут) → получаешь персонажа, который «залочен» во всех генерациях. Технически (higgsfield-ai/skills): Soul ID → `reference_id` → потребляется генерацией через `--soul-id`. Это identity-lock на уровне пайплайна.
- **Storyboard** — родная раскадровка.
- **Кредиты сгорают** в конце месяца. Тарифы (ориентир, сверять): free = 720p + водяной знак + малый запас (на запись не годится) · Basic ~$9 — снимает водяной знак, 1080p · «рабочий» ~$17–29 ≈ сотни кредитов. **Платить помесячно**, не годовой.
- Ограничение лёгкого пути: **фото в Higgsfield заливаешь сам** (агент через обычное подключение это не делает). Снимается на уровне CLI — см. «Потолок».

### Канал 2 — Replicate (по факту за токены, путь практика)
- Ключ + **лимит трат сразу** (правило безопасности). Оплата по факту центами, **баланс не сгорает**. cold start модели ~15–20 сек (первая генерация «думает»).
- Один ключ → весь модельный ряд:
  - **видео:** Kling 3.0, Seedance, Veo 3.1, Wan;
  - **картинки:** nano-banana-pro, Flux;
  - **своё лицо — Flux LoRA:** обучаешь собственную модель на ~15–30 фото → она генерит именно тебя (аналог Soul ID, собранный руками через ключ);
  - апскейл, удаление фона, пакетная генерация 10–50 вариантов.
- Цены (ориентир, сверять на дату): Kling ~$0.07/сек · Seedance ~$0.036/сек · Veo 3.1 Fast ~$0.10/сек / Full ~$0.40/сек · nano-banana-pro $0.134/картинка (через сторонние дешевле). На все тесты урока $10–20 баланса хватает с запасом.

## Image-модели для опорных кадров (Фаза 5)

- **nano-banana-pro** — держит до 14 фото-референсов, консистентное лицо/товар, надёжная кириллица в кадре, **умеет 9:16**. Для вертикальных роликов и надёжного лока лица — **основной выбор**.
- **gpt-image-2** (через Codex по подписке / по ключу) — дешёвый движок. Референс **принимает** через `-i` (товар и простое лицо держит прилично), но **форматы ограничены: 1024×1024 / 1536×1024 / 1024×1536 — максимум вертикали 2:3, 9:16 НЕ умеет.** Годится для черновиков, локаций, горизонтальных/квадратных кадров.

⚠️ **Два подводных камня канала (выявлено пилотами — обязательно учесть):**
1. **Формат под вертикаль.** Для 9:16 (Reels/Shorts/TikTok) **бери nano-banana-pro с явным `aspect_ratio: 9:16`.** Codex/gpt-image 9:16 не отдаёт (максимум 2:3 = 1024×1536) — «указать явно» не спасает, такого размера у него просто нет. Если твой канал по умолчанию рисует через gpt-image/Codex — для раскадровки в вертикаль **переопредели на nano-banana**.
2. **Лок лица/товара — всегда через референс-картинку, не словами.** Референс на вход принимают оба: nano-banana-pro (до 14 фото, сильнее) и Codex `-i` (слабее, но товар/простое лицо держит). Чистого описания словами для консистентности мало — всегда подавай картинку-референс ассета. Под строгий лок лица между многими кадрами — nano-banana-pro / Soul ID.

## Качество видео-моделей (под выбор осознанно)

- **Seedance 2.0** — лидер по качеству (единый аудио-видео-движок).
- **Kling 3.0** — 4K/60fps, Motion Brush, first/last frame.
- **Veo 3.1** — единственная с нативной синхронной речью.
- Дешёвые/быстрые модели слабее по эмоциям и логике движения; топовые — заметно дороже.

**Правило выбора (задача → модель)** — не оставляй выбор «на потом», промт обязан назвать модель:

| Что нужно | Бери |
|---|---|
| плавные переходы / frame chaining (start+end frame) | Kling 3.0 / Veo 3.1 / Luma |
| речь, синхронные губы в кадре | Veo 3.1 |
| максимум качества картинки и движения, бюджет есть | Seedance 2.0 |
| быстро и дёшево пощупать | дешёвые/быстрые (качество ниже) |

Не уверен — уточни у пользователя задачу и подбери по таблице.

## Безопасность ключа (higgsfield-ai/skills)

- Ключ **в чат не вставляем** — вход через device-flow (`auth login`) либо переменную окружения; креды хранятся в конфиге, не в переписке.
- Не обходить абстракцию «голым curl» — обёртка/CLI сама делает авторизацию, повторы, загрузку, валидацию.
- На своём ключе всегда стоит лимит трат.

## Потолок — Higgsfield CLI + UUID (не база)

- **CLI умеет то, чего не умеет лёгкое подключение:** читает файлы с компа и **загружает твои референсы прямо в генерацию** (лицо, товар, окружение) — снимает ручной шаг с загрузкой фото. Сам скачивает готовое и раскладывает по папкам, работает по скрипту/CSV.
- **UUID** — при загрузке референса Higgsfield кодирует его в идентификатор; агент ссылается на референс по UUID без повторной загрузки → **экономия кредитов**.
- Порог выше (терминал, установка) — подавать как «куда расти», не как базу.
