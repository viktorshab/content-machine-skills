# Паспорт движков и каналов генерации

> Какими движками генерим и через какой канал их вызываем. Канал — **сменное звено**: методика конвейера одна, движок внутри меняется. Все цифры (цены, лимиты, кредиты) — **сверять с доками на день работы**, они меняются часто (особенно Higgsfield).

## Как вызываем движки

Генерацию вызывай через **свой подключённый канал** — Higgsfield (MCP) или Replicate (по ключу), как настроено в вашем проекте. Скилл не дёргает API руками: он просит агента сгенерить через подключённый канал, агент сам идёт в нужную модель.

## Два канала (выбираешь под себя)

### Канал 1 — Higgsfield (готовое «под ключ»)
- Подключение одной командой, вход через браузер (OAuth), без ключей:
  `claude mcp add --transport http --scope user higgsfield https://mcp.higgsfield.ai/mcp`
- За одной точкой — 30+ моделей; командуешь «сгенери в такой-то» → агент маршрутизирует.
- **Soul ID** — обучаешь своё лицо (~20 фото, обучение пара минут) → получаешь персонажа, который «залочен» во всех генерациях. Технически (higgsfield-ai/skills): Soul ID → `reference_id` → потребляется генерацией через `--soul-id`. Это identity-lock на уровне пайплайна.
- **Storyboard** — родная раскадровка.
- **Кредиты сгорают** в конце месяца. Тарифы (ориентир, сверять): free = 720p + водяной знак + малый запас (на запись не годится) · Basic ~$9 — снимает водяной знак, 1080p · «рабочий» ~$17–29 ≈ сотни кредитов. **Платить помесячно**, не годовой.
- Ограничение лёгкого пути: **фото в Higgsfield заливаешь сам** (агент через обычное подключение это не делает). Снимается на уровне CLI — см. «Потолок».

### Канал 2 — Replicate (по факту за токены, путь практика)
- Ключ + **небольшой предоплаченный баланс без авто-пополнения** (это и есть потолок расходов — отдельной настройки лимита трат у Replicate больше нет, убрана). Оплата по факту центами, **баланс не сгорает**. cold start модели ~15–20 сек (первая генерация «думает»).
- Один ключ → весь модельный ряд:
  - **видео:** Kling 3.0 (своё лицо — `kling-v3-omni-video`, берёт 2–4 фото-референса и лочит лицо Element Binding'ом, дрейф <10%; базовый i2v — `kling-v3-video`), Seedance, Veo 3.1, Wan;
  - **картинки:** nano-banana-pro, Flux;
  - **своё лицо — Flux LoRA:** обучаешь собственную модель на ~15–30 фото → она генерит именно тебя (аналог Soul ID, собранный руками через ключ);
  - апскейл, удаление фона, пакетная генерация 10–50 вариантов.
- Цены (ориентир, сверять на дату): Kling ~$0.07/сек · Seedance ~$0.036/сек · Veo 3.1 Fast ~$0.10/сек / Full ~$0.40/сек · nano-banana-pro $0.134/картинка (через сторонние дешевле). На все первые тесты $10–20 баланса хватает с запасом.

## Image-модели для опорных кадров (Фаза 5)

- **gpt-image-2 (Image 2)** — **основной движок для лица.** По реальному тесту (2026-06-18) лицо ученика держит именно он (6 фото-референсов через `-i` + полный landmark-lock + анти-beauty); Flux-reference на лице провалился — «не он». Референс принимает через `-i`. Под лицо вшивай полный identity-блок (`03-prompts.md`). Id модели для прямого вызова через OpenAI API — `gpt-image-2-2026-04-21`.
  - **Размеры — внимательно, тут частая ошибка.** Через ChatGPT/Codex максимум вертикали — **2:3 = 1024×1536** (есть ещё 1024×1024 и 1536×1024). Настоящий **9:16 у Image 2 есть, но ТОЛЬКО через OpenAI API напрямую**: валидные размеры `1152x2048` и `1440x2560` (каждая сторона кратна 16; `1080x1920` **невалиден** — 1080 не делится на 16). На Replicate прямого 9:16 нет — ближайший вертикальный `aspect_ratio: 2:3`.
  - **Формат задавай параметром `size`/`aspect_ratio`, НЕ суффиксом `9:16` в тексте промта** — словами в промте формат не форсится (не Midjourney-суффикс).
- **nano-banana-pro** (Google Gemini 3 Pro Image, `google/nano-banana-pro` / `gemini-3-pro-image-preview`) — тоже умеет лицо (до 6 референсов), держит товар, надёжная кириллица в кадре, нативный `aspect_ratio: 9:16`. ⚠️ При референс-фото на входе модель уходит в режим редактирования и **наследует размеры исходного фото** — вертикаль 9:16 форсируй тремя слоями: (1) `aspect_ratio:"9:16"` явно; (2) дубль словами в промте; (3) **canvas-трюк — загрузи пустой прозрачный PNG 9:16 как референс и скажи `fill the entire canvas`** (важнее всего). `resolution:"2K"`. В нашем опыте на лице — **«с возни»**, поэтому под лицо запасной вариант (основной — Image 2); но именно он даёт нативный 9:16 без выхода в OpenAI API.

⚠️ **Два подводных камня канала (выявлено пилотами — обязательно учесть):**
1. **Формат под вертикаль.** Если генеришь через ChatGPT/Codex — потолок вертикали `1024×1536` (2:3), 9:16 там нет. Настоящий 9:16 у Image 2 доступен **через OpenAI API напрямую** (`size="1152x2048"` / `"1440x2560"`, сторона кратна 16, `1080x1920` невалиден). Не пытайся форсить 9:16 суффиксом в промте — он задаётся параметром размера. Нужен нативный 9:16 без API — можно взять nano-banana-pro с `aspect_ratio: 9:16` (но на лице он «с возни»).
2. **Лок лица/товара — всегда через референс-картинку, не словами.** Референс на вход принимают оба: Image 2 `-i` (основной под лицо) и nano-banana-pro (до 6 фото). Чистого описания словами для консистентности мало — всегда подавай картинку-референс ассета + полный identity-блок.

## Качество видео-моделей (под выбор осознанно)

- **Kling 3.0** — **основная под своё лицо.** `kling-v3-omni-video` принимает 2–4 фото-референса и держит лицо (Element Binding — лицо как 3D-якорь, дрейф упал с 50%+ до <10%); `kling-v3-video` — базовый i2v (start+end frame, negative_prompt, pro=1080p, 3–15с). 4K/60fps, Motion Brush.
- **Seedance 2.0** — лидер по «вау»-качеству, НО реальное распознаваемое лицо **режет на уровне модели** (refusal E005) — под свой ролик с лицом непригоден.
- **Veo 3.1** — единственная с нативной синхронной речью; реалистичные неавторизованные лица может резать — бери только под кадры без задачи сохранить лицо.
- **Fallback под лицо** — `xai/grok-imagine-video-1.5` (свежая, мало проверена на удержание конкретного лица — сравни на боевом кадре).
- Дешёвые/быстрые модели слабее по эмоциям и логике движения; топовые — заметно дороже.

**Правило выбора (задача → модель)** — не оставляй выбор «на потом», промт обязан назвать модель:

| Что нужно | Бери |
|---|---|
| плавные переходы / frame chaining (start+end frame) | Kling 3.0 / Veo 3.1 / Luma |
| речь, синхронные губы в кадре | Veo 3.1 |
| максимум качества картинки и движения, бюджет есть | Seedance 2.0 |
| быстро и дёшево пощупать | дешёвые/быстрые (качество ниже) |

Не уверен — уточни у пользователя задачу и подбери по таблице.

### Шпаргалка по видео-моделям (под тип кадра)

Это карта «что чем оживлять» — НЕ канон: версии/цены/живость моделей меняются часто, **сверяй на день работы** (правило файла). Главная колонка — «под какой кадр»: модель берётся под тип кадра, а не «одна лучшая на всё».

| Модель | Управление движением/камерой | start+end frame | Модерация реальных лиц | Под какой кадр |
|---|---|---|---|---|
| **Kling 3.0** (omni — под лицо) | Motion Brush, camera moves, Multi-Shot; `kling-v3-omni-video` берёт 2–4 фото лица | да (Start&End) | лицо держит (Omni — Element Binding); при сильном движении возможен дрейф | лицо героя + динамичный проезд, посадка, пролёт |
| **Veo 3.1** | сильный prompt-камер-контроль, нативная речь | да (first/last) | safety-фильтры могут резать фотореалистичных людей | direction-critical интерполяция, финальный отъезд, речь в кадре |
| **Runway Gen-4/4.5** | простые motion-промты, аккуратный chaining | chaining через последний кадр | для лица — малая амплитуда | close-up лица, атмосферный push-in, деталь |
| **Hailuo (MiniMax)** | явные `[command]` камеры, `prompt_optimizer:false` | да (first&last) | subject-reference по фото лица | контролируемый tracking/push-in, посадка |
| **Seedance 2.0** | топ по качеству, мультимодальный | через keyframe/reference | ⚠ режет реальные лица (ошибка E005) — под персональный ролик не основной | сцены/стилизация, НЕ лицо героя |
| **Luma Ray** | keyframes, camera moves | да (keyframes/start-end) | character reference для консистентности | интерполяция, продление движения, переходы |
| **Wan (локально)** | open-source, ComfyUI | FLF2V | полный локальный контроль | batch-эксперименты, приватность; не быстрый путь к premium |

Точные длительности/разрешения/цены и имена моделей-акторов — на страницах моделей подключённого канала и в доках на день работы (меняются часто). Сходство лица по референсам и continuity направления — отдельные темы, разобраны в `03-prompts.md` и `02-brief-shotlist.md`.

## Безопасность ключа (higgsfield-ai/skills)

- Ключ **в чат не вставляем** — вход через device-flow (`auth login`) либо переменную окружения; креды хранятся в конфиге, не в переписке.
- Не обходить абстракцию «голым curl» — обёртка/CLI сама делает авторизацию, повторы, загрузку, валидацию.
- На своём ключе потолок задаёт **предоплаченный баланс без авто-пополнения** — отдельной настройки лимита трат у Replicate больше нет (убрана). Не включаешь авто-пополнение — потратить заметно больше нельзя.

## Потолок — Higgsfield CLI + UUID (не база)

- **CLI умеет то, чего не умеет лёгкое подключение:** читает файлы с компа и **загружает твои референсы прямо в генерацию** (лицо, товар, окружение) — снимает ручной шаг с загрузкой фото. Сам скачивает готовое и раскладывает по папкам, работает по скрипту/CSV.
- **UUID** — при загрузке референса Higgsfield кодирует его в идентификатор; агент ссылается на референс по UUID без повторной загрузки → **экономия кредитов**.
- Порог выше (терминал, установка) — подавать как «куда расти», не как базу.
